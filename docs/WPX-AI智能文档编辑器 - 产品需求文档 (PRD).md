# AI智能文档编辑器 - 产品需求文档 (PRD)

**版本**：V1.0  

**状态**：MVP定义阶段  

**核心理念**：做一个有身份、有知识、有习惯的AI原生创作空间，覆盖WPS 80%的常用编辑能力，在AI体验上实现降维打击。

---

## 一、产品定位与目标用户

### 1.1 产品定位

面向个人创作者、知识工作者、中小团队的**AI原生文档工作站**。以Markdown为核心编辑格式，AI对话为主要交互方式，集成轻量表格、图片处理、智能文库，实现从创作、编辑到知识管理的完整闭环。

### 1.2 目标用户

- 自媒体作者、博主

- 技术文档编写者

- 产品经理、项目经理

- 对WPS付费不满、追求效率的普通办公用户

### 1.3 核心价值主张

- **数据主权**：文档格式开放（MD标准），随时导出，不被锁定

- **AI原生**：自然语言驱动编辑、排版、图片处理、知识检索

- **越用越懂你**：学习用户习惯，自动生成个性化模板和分类规则

- **轻量免费**：纯Web端，零客户端占用，PDF转Word等核心功能免费

---

## 二、产品架构全景图

产品由五大模块构成一个创作与管理的正向循环：

```

┌─────────────────────────────────────────────────┐

│                 AI 助手（右下角头像）              │

│           对话驱动所有操作，非侵入式交互            │

└────────────────────┬────────────────────────────┘

                     │

     ┌───────────────┼───────────────┐

     │               │               │

┌────▼────┐   ┌──────▼──────┐   ┌───▼────┐

│ 文档编辑 │   │  轻量表格   │   │ 图片处理│

│ (Tiptap)│   │(Tiptap扩展) │   │(图片编辑)│

└────┬────┘   └──────┬──────┘   └───┬────┘

     │               │               │

     └───────────────┼───────────────┘

                     │

     ┌───────────────┼───────────────┐

     │               │               │

┌────▼────┐   ┌──────▼──────┐   ┌───▼────┐

│资料库   │   │  记忆与模板  │   │智能文库 │

│(RAG检索)│   │(习惯学习)   │   │(自动归档)│

└─────────┘   └─────────────┘   └────────┘

```

**各模块关系**：

- **底座**（文档编辑+表格+图片处理）提供创作能力

- **外脑**（资料库）为AI提供事实依据，解决幻觉问题

- **记忆**（习惯与模板）让AI适配用户个人风格

- **成果库**（智能文库）自动整理产出，盘活知识资产

- **大脑**（AI助手）统一调度上述所有能力

## 三、功能需求详述

### 3.1 核心编辑模块

#### 3.1.1 文档编辑器

| 需求项 | 描述 | 优先级 |

|:---|:---|:---|

| Markdown编辑 | 所见即所得，支持常用Markdown语法 | P0 |

| 富文本能力 | 字体、字号、颜色、加粗斜体、行距、对齐 | P0 |

| 段落样式 | 标题H1-H6、引用、列表、分隔线、代码块 | P0 |

| 图片插入 | 支持粘贴、拖拽、本地上传、URL链接 | P0 |

| 查找替换 | 支持文本查找与替换 | P1 |

| 撤销重做 | 完整的历史记录支持 | P0 |

**不做**：复杂合并、域代码、宏、公文排版、邮件合并

#### 3.1.2 轻量表格

| 需求项 | 描述 | 优先级 |

|:---|:---|:---|

| 基础表格 | 插入N行M列表格，行列增删 | P0 |

| 合并拆分 | 单元格合并与拆分 | P1 |

| 样式调整 | 边框、背景色、文字对齐 | P1 |

| 简单排序 | 按列排序 | P2 |

| AI建表 | 对话描述需求，AI生成表格 | P1 |

**不做**：公式计算、数据透视、图表、高级筛选（这些属于独立电子表格范畴）

#### 3.1.3 图片轻处理

| 需求项 | 描述 | 优先级 |

|:---|:---|:---|

| 裁剪 | 自由比例、固定比例（1:1/4:3/16:9） | P0 |

| 旋转翻转 | 90°旋转、水平/垂直翻转 | P0 |

| 基础调色 | 亮度、对比度、饱和度、色温 | P1 |

| 标注工具 | 箭头、矩形框、圆形框、文字标注 | P1 |

| 马赛克/模糊 | 涂抹式打码，保护敏感信息 | P1 |

| AI一键去背景 | 自动识别主体，去除背景 | P0（杀手功能） |

| AI自然语言修图 | “把背景调亮”“给手机号打码” | P1 |

**不做**：图层、蒙版、曲线、通道、高级滤镜参数

### 3.2 AI助手交互系统

#### 3.2.1 助手入口

| 需求项 | 描述 | 优先级 |

|:---|:---|:---|

| 右下角头像 | 固定位置，点击唤起对话窗，可换头像 | P0 |

| 浮动对话窗 | 基于vue3-draggable-resizable实现 | P0 |

| 可拖拽 | 自由拖动位置，不遮挡编辑区 | P0 |

| 可缩放 | 最小300x300，支持边缘拖拽调整大小 | P0 |

| 钉住功能 | 点击📌按钮固定窗口，防止误关闭 | P0 |

#### 3.2.2 AI能力范围

| 能力 | 触发方式 | 描述 | 优先级 |

|:---|:---|:---|:---|

| 生成初稿 | 对话 | “写一份Q3运营计划”，AI生成结构化MD文档 | P0 |

| 选中修改 | 选中文本后对话 | “精简到3个要点”“改成更正式的语气” | P0 |

| 全文改写 | 对话 | “全文润色，保持原意”“翻译成英文” | P1 |

| 表格对话 | 选中表格后对话 | “加一列备注”“把价格列右对齐” | P1 |

| 图片对话 | 对话中引用图片 | “去背景”“把文字放大加粗” | P1 |

| 资料检索写作 | @引用资料库 | “根据技术文档第三章，写个快速入门” | P1 |

### 3.3 资料库与RAG检索

| 需求项 | 描述 | 优先级 |

|:---|:---|:---|

| 资料上传 | 支持PDF/Word/MD/网页链接/纯文本粘贴 | P1 |

| 自动解析 | 提取正文，切片，向量化存储 | P1 |

| 侧边栏管理 | 资料列表，可预览、删除、搜索 | P1 |

| @引用 | 对话中输入@选择资料作为参考源 | P1 |

| 写作自动检索 | 写作时AI自动关联相关资料片段 | P2 |

### 3.4 用户记忆与智能模板

| 需求项 | 描述 | 优先级 |

|:---|:---|:---|

| 习惯采集 | 自动记录字体、字号、行距、标题样式偏好 | P1 |

| 结构学习 | 识别用户某类文档的固定结构（如周报三段式） | P1 |

| 用语学习 | 记录高频词汇、签名语、称呼习惯 | P2 |

| 智能模板生成 | 同类文档写3-5次后，自动生成专属模板 | P1 |

| 新建时推荐 | 新建文档时优先展示“我的智能模板” | P1 |

| 一键套用 | 对话说“用我的标准周报模板写” | P1 |

### 3.5 智能文库

| 需求项 | 描述 | 优先级 |

|:---|:---|:---|

| 根目录指定 | 用户指定一个本地文件夹或云端路径作为文库根 | P1 |

| 自动分类 | AI分析文档内容，自动归类到子文件夹 | P1 |

| 分类建议 | 保存时弹出AI建议路径，用户确认或修改 | P1 |

| 规则学习 | 多次手动调整后，AI更新分类规则 | P2 |

| Wiki式浏览 | 生成目录树、标签云、自动索引页 | P2 |

| 智能链接 | 文档间根据引用关系自动互链 | P2 |

| 全文搜索 | 跨文档全文检索，支持AI问答式搜索 | P2 |

| 版本管理 | 同名文档自动保留历史版本 | P2 |

## 四、多格式导出能力

| 导出格式 | 实现方式 | 描述 | 优先级 |

|:---|:---|:---|:---|

| Markdown | 原生 | 保持原始MD格式 | P0 |

| PDF | Pandoc | 高质量PDF，可带书签 | P0 |

## 五、非功能需求

### 5.1 性能要求

- 首屏加载 < 3秒

- 文档打开（1万字以内）< 1秒

- AI对话响应流式输出，首字延迟 < 2秒

- 图片处理操作延迟 < 500ms

### 5.2 兼容性

- 支持 Chrome / Edge / Safari / Firefox 主流浏览器最新两个版本

- 优先适配桌面端（1920x1080 / 1440x900），移动端后期考虑

### 5.3 安全与隐私

- 用户文档默认本地浏览器处理，不上传服务器

- AI功能需明确告知数据用于模型推理

- 记忆库和资料库数据加密存储

- 提供数据导出和账户删除功能

## 六、MVP迭代路线图

### Phase 1：核心文档编辑器（第1-4周）

**目标**：能在浏览器里写Markdown，导出Word/PDF，AI能改文章

**交付物**：

- [ ] Tiptap编辑器搭建，支持基础Markdown和富文本

- [ ] AI头像+浮动对话窗，选中文本对话修改

- [ ] Pandoc导出服务（MD/HTML/Word/PDF）

- [ ] 产品首页和基础UI

**验收标准**：可写文章→AI改→导出Word，全流程跑通

### Phase 2：表格与图片处理（第5-7周）

**目标**：补全常用内容类型，图片功能形成差异化

**交付物**：

- [ ] Tiptap表格扩展集成

- [ ] tui.image-editor 图片编辑器集成

- [ ] AI去背景API接入

- [ ] 图片编辑与文档编辑无缝流转

**验收标准**：文档内可插表格、可编辑图片、可AI去背景

### Phase 3：记忆与文库（第8-10周）

**目标**：从工具升级为知识平台，形成用户粘性

**交付物**：

- [ ] 资料库上传、解析、向量化

- [ ] 用户习惯采集与智能模板生成

- [ ] 智能文库自动分类与归档

- [ ] @引用资料写作能力

**验收标准**：AI能基于用户资料写作，文档自动归档，智能模板可用

### Phase 4：打磨与优化（第11-12周）

**目标**：体验打磨，准备公开发布

**交付物**：

- [ ] Wiki式浏览界面

- [ ] 全文搜索

- [ ] 性能优化与BUG修复

- [ ] 用户引导和帮助文档

## 七、技术选型

| 层级 | 技术 | 说明 |

|:---|:---|:---|

| 前端框架 | Vue 3 | 与PPTist技术栈一致，便于后续扩展 |

| 编辑器核心 | Tiptap | 社区活跃，插件生态全，JSON驱动适配AI |

| 表格 | Tiptap Extension Table | 编辑器原生集成 |

| 图片编辑器 | tui.image-editor | 开源，功能覆盖80%需求 |

| 拖拽组件 | vue3-draggable-resizable | AI对话窗和图片编辑窗的拖拽缩放 |

| UI库 | shadcn/ui (Vue版) | 无样式捆绑，方便定制 |

| AI SDK | Vercel AI SDK | 流式响应，前端集成方便 |

| 大模型 | DeepSeek API | 性价比高，国内可用 |

| 文档转换 | Pandoc | 部署为Node服务 |

| AI去背景 | rembg | Python服务，U²-Net模型 |

| 文档解析 | Tika / Unstructured | 资料库文档解析 |

| 向量存储 | Chroma | 轻量级，适合MVP |

| 工作流编排 | DeerFlow | 记忆学习、RAG检索、自动分类等复杂工作流 |

| 全文搜索 | Meilisearch | 轻量高效，部署简单 |

## 九、风险与应对

| 风险 | 影响 | 应对策略 |

|:---|:---|:---|

| Pandoc兼容性 | 复杂排版导出失真 | 仅承诺标准格式导出，不追求100%还原 |

| AI API成本 | 运营成本高 | 优先用DeepSeek等性价比模型，高频操作本地缓存 |

| 用户隐私担忧 | 影响信任 | 本地优先处理，明确数据使用边界，提供离线模式 |

| 竞品跟进 | WPS加AI功能 | 聚焦差异化（习惯学习+知识库），建立数据护城河 |
