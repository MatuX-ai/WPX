## 〇、业务模式变更说明（V1.0 → V1.1）

> **重要变更**：自 V1.1 起，WPX 字体库业务调整为：

1. **取消「商业字体商店」与「Token 充值」**：不再提供平台端的商业字体服务（不再提供方正、汉仪、华康等商业字体下载与授权服务）；不再提供「Token 充值」「按字计费」机制。
2. **保留 8 款免费内置字体**：思源黑体/宋体、霞鹜文楷、阿里巴巴普惠体、HarmonyOS Sans、JetBrains Mono、Noto Color Emoji 随安装包提供，**永久免费**。
3. **保留「在线免费字体」按需下载**：站酷快乐体、得意黑等 9 款免费可商用字体仍可一键下载使用。
4. **保留「导入本地字体」**：用户可在「设置 → 字体管理 → 导入本地字体」中选择已采购授权的本地字体文件（.ttf / .otf / .ttc / .woff2），直接加载到编辑器；该功能**不**产生任何平台 Token 消耗。
5. **「导出嵌入商业字体」不再存在**：用户商业字体仅在本机使用，导出时由用户自行保证商用授权；**WPX 不嵌入、不计费、不追踪商业字体使用情况**。
6. **后端接口下线**：`/api/token/*`、`/api/font-market/*`、`/api/font-consume/*` 等接口已废弃，前端对应 UI 全部禁用。

本文档下述章节已按上述变更同步更新；原商业字体 Token 计费章节标记为「已废弃」仅供历史参考。

## 1. 概述

为 WPX 构建**完全免费**的字体库系统。**仅**包含内置免费商用字体 + 在线免费字体库；商业字体需求由用户自行采购合法授权后通过「导入本地字体」使用。

**核心原则**：

- 全部内置 / 在线字体 100% 免费可商用。
- 商业字体不卖、不收、不嵌入计费；用户自负责。
- 不再需要 Token、充值、扣费等任何机制。

### 3.5 商业字体的替代方案（V1.1 新增）

由于平台不再提供商业字体，用户可采用以下方式使用商业字体：

1. **采购授权后导入本地字体**
   - 在方正、汉仪、华康、蒙纳、TypeTogether 等官方渠道购买个人 / 商用授权。
   - 在 WPX 桌面客户端「设置 → 字体管理 → 导入本地字体」中选择 `.ttf` / `.otf` / `.ttc` / `.woff2` 文件，导入后立即在字体下拉框中出现。
   - 导入的字体仅在**本机**的编辑器与导出中使用，**不**上传 WPX 服务器，**不**嵌入 PDF / DOCX（避免未授权分发风险）。

2. **使用免费开源字体替代**
   - 8 款内置字体（思源黑体 / 宋体、霞鹜文楷、阿里巴巴普惠体、HarmonyOS Sans 等）已覆盖绝大多数文档场景。
   - 9 款在线免费字体（站酷快乐体、得意黑等）按需下载，覆盖标题、海报等装饰场景。

3. **商用授权合规责任**
   - 商业字体的授权合规由用户自行负责；WPX 不提供、不验证、不追踪任何商业字体的授权状态。

> 以下内容为 V1.0 时期的旧表，已被本章 3.5 替代方案取代，仅供历史参考，请勿实现。

| 充值方式（已废弃） | 金额（已废弃） | 获得 Token（已废弃） | 说明（已废弃） |

|:---|:---|:---|:---|

| **10 元包** | ¥10 | 200 Token | 默认推荐，满足多数场景 |

| **自定义充值** | ¥N（N 为 10 的整数倍） | N × 20 Token | 最低 ¥10，上不封顶 |

**换算**：1 元 = 20 Token，1 个汉字导出收费 = 0.05 元。

### 3.2 自定义充值规则

- 输入框默认值为 10，步进为 10，最小值为 10，无上限。

- 实时显示预览：“充值 ¥30 可得 600 Token”。

- 支付方式：微信/支付宝。

### 3.3 Token 消耗规则

| 操作 | 消耗 Token | 备注 |

|:---|:---|:---|

| 编辑器内预览商业字体 | 免费 | 随时看效果 |

| 导出文档中使用商业字体的每个汉字 | 1 Token/字 | 英文字母、数字、标点不收费 |

| 同一文档同一字体 7 天内重复导出 | 不扣 | 后端校验内容哈希 |

| 文档内同时使用多款商业字体 | 分别计费 | 如用两款字体各 20 字，扣 40 Token |

- Token **永久有效**，不过期，不清零。

- 用户可在“我的字体”页面查看余额和使用明细。

## 5. 核心交互流程

### 5.1 充值流程

```

用户进入字体商店

    → 看到余额显示“我的 Token：XXX”

    → 选择“10 元包”或输入“自定义金额”

    → 点击“立即充值”

    → 弹出支付二维码（微信/支付宝）

    → 支付成功，Token 实时到账

    → 显示充值成功 Toast + 新余额

```

## 8. 技术实现要点

### 8.1 字体文件管理

- **免费内置字体**：打包在 `resources/fonts/built-in/`，子集化处理。

- **在线免费字体**：存储在 CDN，用户点击下载后存到 `userData/fonts/free/`。

- **商业字体**：加密存储在 `userData/fonts/commercial/`，密钥与用户账户绑定。

### 8.2 字体加密与解密

- 商业字体使用 AES-256 加密存储。

- 解密密钥从服务端获取，每次启动应用时刷新临时令牌。

- 渲染进程不直接访问字体文件，主进程解密后注入显示。

### 8.3 导出时的字体嵌入

- 导出时，主进程对商业字体执行子集化提取（仅取文档中实际用到的字符）。

- 子集嵌入 PDF/DOCX，确保接收方无需安装该字体即可查看。

- 嵌入的子集包含用户 ID 水印，可追溯盗版。

### 8.4 Token 管理系统

后端 API：

- `POST /api/token/recharge`：创建充值订单，返回支付链接。

- `GET /api/token/balance`：查询余额。

- `POST /api/token/consume`：导出时扣费，校验余额并返回结果。

- `GET /api/token/records`：消费记录列表。
