1. 设计原则

沉浸式编辑：编辑器永远居中最大化，无侧边栏干扰

AI助手随时在位，但不霸占舞台：浮动窗口形态，默认收起

一键直达：所有操作不超过2次点击

尊重原生：适配Windows/macOS原生窗口控件

2. 全局布局与窗口模型

主窗口：无边框自定义标题栏（或保留系统标题栏？抉择与理由）

左上角：文档名 + 自动保存状态指示灯

顶部工具栏：仅保留最常用按钮（加粗、斜体、标题、插入图片、插入表格、导出），高度36px

无右侧面板，无底部状态栏（精简）

AI助手浮层体系：

右下角头像 (56x56, fixed, z-index 1000)

对话窗 (默认400x500, 最小300x300, z-index 1001)

图片编辑器 (默认800x600, z-index 1002)

窗口层级关系图

多窗口支持：用户可同时打开多个编辑器窗口（每个窗口独立状态）

3. 桌面端特有交互

系统托盘：最小化到托盘，右键菜单（新建文档、快速笔记、退出）

全局快捷键：Ctrl+Shift+W 呼出AI助手；Ctrl+N 新建；等等

文件关联：双击.md文件直接打开WPX

拖拽能力：从桌面拖入图片/文件到编辑器；从编辑器拖出文字到桌面

右键菜单：编辑器内右键（剪切/复制/粘贴/全选/用AI改写选中内容）

4. 响应式与窗口自适应

窗口最小尺寸 900x600，小于此尺寸时工具栏折叠，AI头像缩小

浮动窗口不得超出主窗口边界（parentLimitation）

5. 主题与视觉规范

默认浅色/深色双主题，跟随系统自动切换

色彩、字体、阴影、圆角、间距的精确值

图标使用lucide图标集，统一描边风格

6. 动效规范

对话窗弹出：scale + fade，200ms ease-out

消息出现：打字机效果或逐条滑入

图片编辑器工具栏切换：平滑过渡

7. 错误与空状态设计

首次打开无文档时的空状态插画与引导

网络断开时AI头像变灰，浮窗提示“当前处于离线模式”

导出失败时的错误提示位置（顶部toast，不打断操作）

8. 无障碍基础要求

支持Tab键导航

足够的颜色对比度

为AI头像提供aria-label
