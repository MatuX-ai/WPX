## 2. 公共大模型配额：按 Token，不按次数

平台向上游采购大模型按 **Token** 计费，WPX 对用户的公共模型免费额度必须与之一致，**禁止**按「每日调用次数」计数。

| 项目 | 规则 |
|:---|:---|
| 计量单位 | Token（优先使用 API 返回的 `total_tokens`，含 prompt + completion） |
| 访客额度 | 0 Token/天 |
| 注册用户额度 | 可配置，默认 100M Token/天，按**用户 ID**累计 |
| 重置周期 | 每日 0 点（本地或服务器时区，与实现一致即可） |
| 扣减时机 | 每次平台模型对话**完成后**，按本次实际 Token 累加 |
| 调用前检查 | 剩余 Token > 0 才允许发起平台模型请求 |
| 额度用尽 | 未配置自定义 Key → 引导「去配置」自有 API；已配置 → 充值或次日 |

**实现参考**：`electron/services/free-quota-store.js`、`wpx-app/src/utils/freeQuota.js`、`useAiChat.js`（`onFinish` 扣 Token）。

## 4. 与字体 Token 的区别

| 类型 | 用途 | 计量 |
|:---|:---|:---|
| **公共大模型 Token 额度** | AI 对话调用 WPX 提供的模型 | 按模型 API usage 扣减，每日免费池 + 可充值 |
| **字体 Token** | 商业字体导出按字扣费 | 1 汉字 = 1 Token，永久有效 |

两者均为「Token」，但账户与扣减逻辑独立，需求与实现勿混用。
