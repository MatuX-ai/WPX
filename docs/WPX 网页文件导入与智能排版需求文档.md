## 1. 概述

为 WPX 增加网页文件（HTML）导入能力，并设计智能排版触发时机。与 Markdown 文件“导入即排版”不同，网页文件采用“导入保留原样，A4 阅读模式触发排版”的策略，以符合 WPX“编辑无纸张，导出有边界”的核心理念。

**核心原则**：
- 导入时保留 HTML 原始视觉结构，不主动破坏网页设计意图
- 排版模板选择仅在用户有明确输出意图时触发
- 原始 HTML 源码存入文档隐藏属性，供排版引擎随时调用
- 用户可随时在“原样”和“排版后”之间切换

---

## 2. 为什么 HTML 和 MD 的排版触发时机不同

| 对比维度 | Markdown 导入 | HTML 导入 |
|:---|:---|:---|
| **源内容形态** | 纯文本+标记符号（`#`、`-`、`|`） | 已渲染的富文本网页 |
| **用户看到的样子** | 满屏标记符号，阅读困难 | 完整视觉结构，可直接阅读 |
| **用户意图** | 想用这个内容继续写作 | 想收藏、存档、阅读或提取内容 |
| **排版触发时机** | 导入后立即触发 | 切换到 A4 阅读模式时触发 |
| **排版前状态** | 不可读的源码 | 可读的网页原样 |

## 3. 功能流程

### 3.1 导入阶段

```
用户拖入HTML文件 / 粘贴网页内容 / 粘贴URL
    ↓
WPX 检测到 HTML 内容
    ↓
编辑器渲染网页原样
    ↓
原始 HTML 源码存入文档 attrs.htmlSource
    ↓
用户正常浏览、编辑
    ↓
无任何弹窗或提示（保持纯净）
```

### 3.2 A4 阅读模式触发排版

```
用户点击“焦点模式”或切换到 A4 阅读模式
    ↓
WPX 检测当前文档是否有 attrs.htmlSource
    ↓
AI 助手弹出排版模板选择框
    ↓
用户选择模板 → 排版引擎读取原始 HTML → 应用模板规则 → 渲染为规范文档
    ↓
用户选择“保持原样” → 继续以网页原样展示
```

### 3.3 排版后切换

```
排版完成后
    ↓
用户说“恢复原样”或点击恢复按钮
    ↓
从 attrs.htmlSource 重新渲染为网页原样
    ↓
排版结果丢弃，用户可再次选择模板
```

## 4. HTML 源码缓存方案

### 4.1 存储位置

原始 HTML 存入 Tiptap 编辑器文档 JSON 的 `attrs` 字段中：

```json
{
  "type": "doc",
  "attrs": {
    "htmlSource": "<!DOCTYPE html><html>...完整原始HTML...</html>",
    "sourceUrl": "https://example.com/article",
    "importedAt": "2026-06-26T10:30:00Z",
    "importSource": "paste"
  },
  "content": [
    // Tiptap 解析后的节点树
  ]
}
```

### 4.2 字段说明

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| `htmlSource` | string | 完整原始 HTML 源码 |
| `sourceUrl` | string\|null | 来源 URL（粘贴 URL 导入时记录） |
| `importedAt` | string | ISO 8601 格式的导入时间戳 |
| `importSource` | string | 导入方式：`paste`（粘贴内容）、`file`（拖入文件）、`url`（粘贴链接导入） |

### 4.3 缓存生命周期

| 事件 | 处理 |
|:---|:---|
| 用户导入 HTML | 写入 `htmlSource` |
| 用户确认排版并保存 | 保留 `htmlSource`（方便后续恢复原样） |
| 用户手动“清除格式” | 清除 `htmlSource` 及所有 HTML 相关 attrs |
| 用户从未进入 A4 模式 | 始终保留，直到文档被删除 |
| 导出 MD/PDF/Word | 不导出 `htmlSource` 等内部属性 |
| 文档发送给他人 | `htmlSource` 随文档一起传输，排版能力也跟着走 |

## 5. 排版模板

复用 MD 智能排版引擎的 6 个模板，新增 1 个 HTML 专用模板：

| 模板 | 适用场景 | 说明 |
|:---|:---|:---|
| 通用文章 | 日常写作、自媒体 | 标准段落排版 |
| 正式报告 | 商务汇报 | 标题层级分明 |
| 公文通知 | 机关文件 | 仿红头文件风格 |
| 教案模板 | 教学设计 | 结构化教育模板 |
| 科研论文 | 学术写作 | 学术规范排版 |
| **网页存档** | 收藏网页 | **新增**：保留网页基本信息（来源URL、抓取时间），正文按标准排版 |
| 保持原样 | 仅浏览 | 不触发排版，继续以网页原样展示 |

## 6. 交互设计

### 6.1 导入时（无弹窗）

用户粘贴网页内容或拖入HTML文件后，编辑器静默渲染，无任何提示。右下角 AI 头像旁边显示半透明小字“网页已导入”，3 秒后自动消失。

### 6.2 A4 阅读模式触发时（排版选择弹窗）

```
┌──────────────────────────────────────┐
│  📄 检测到网页内容                     │
│                                      │
│  来源：https://example.com/article   │
│  导入时间：2026-06-26 10:30           │
│                                      │
│  进入 A4 阅读模式，是否按模板排版？      │
│                                      │
│  [通用文章] [正式报告] [公文通知]      │
│  [教案模板] [科研论文] [网页存档]      │
│  [保持原样]                           │
│                                      │
│  ☐ 图片自适应页面宽度                 │
│  ☐ 生成页眉（含来源URL）              │
│                                      │
│  [确认排版]                           │
└──────────────────────────────────────┘
```

### 6.3 排版完成后（恢复入口）

```
编辑器顶部或底部出现轻量提示条：

"✅ 已按【正式报告】格式排版  [恢复原样]  [换模板]  [✕]"
```

- 点击“恢复原样”：从 `htmlSource` 重新渲染网页原样
- 点击“换模板”：重新弹出模板选择框
- 点击 ✕：关闭提示条，保持当前排版

## 7. 技术实现

### 7.1 新增/修改文件

| 文件 | 用途 |
|:---|:---|
| `src/composables/useHtmlImporter.js` | HTML 导入处理：渲染 + 缓存源码 |
| `src/composables/useHtmlFormatter.js` | HTML 排版引擎：读取源码 + 应用模板 |
| `src/composables/useMarkdownFormatter.js` | 扩展：增加 HTML 排版触发逻辑 |
| `src/components/share/FormatTemplateSelector.vue` | 排版模板选择弹窗（MD和HTML共用） |

### 7.2 HTML 排版引擎核心逻辑

```javascript
// useHtmlFormatter.js
function formatHtmlDocument(template, htmlSource) {
  // 1. 解析原始 HTML，提取结构化内容
  const parsed = parseHTML(htmlSource);
  
  // 2. 根据模板规则映射：
  //    - H1 → 一级标题（按模板规则设置对齐、字号、加粗）
  //    - H2 → 二级标题
  //    - <table> → Tiptap 表格节点
  //    - <img> → Tiptap 图片节点（按图片规则调整尺寸和对齐）
  //    - <p> → 正文段落（按模板规则设置首行缩进、行距）
  
  // 3. 剥离原有 CSS 样式，替换为模板样式
  
  // 4. 特殊处理：
  //    - 保留原文中的链接（转为 Tiptap 链接标记）
  //    - 保留原文中的列表结构
  //    - 代码块保留原始格式
  
  return formattedDocument;
}
```

### 7.3 与现有 MD 排版引擎的关系

```
排版请求
    ↓
检测文档类型
    ├── attrs.htmlSource 存在 → 调用 useHtmlFormatter
    ├── 文档含 MD 标记 → 调用 useMarkdownFormatter  
    └── 普通文档 → 直接应用模板规则
```
