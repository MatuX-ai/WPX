## WPX MD 智能排版引擎需求文档

**版本**：V1.1（已实现）  
**状态**：已实现 · useMarkdownFormatter + FormatTemplateSelector + LocalCommandMessage 全部就绪  
**关联文档**：AI 本地指令系统、虚拟纸张与导出母版、[技术架构总览 V2.0](./WPX%20技术架构总览%20V2.0.md)  
**最后更新**：2026-06-28

### 1. 核心价值

当用户导入或粘贴含 MD 标记的文本时，WPX 不再让它保持“源码”形态，而是自动识别并提供排版建议。用户选择一个模板，编辑器立即完成格式清洗和结构化排版，整个过程零延迟、零 Token 消耗。

### 2. 交互流程

1.  **触发检测**：用户粘贴或导入文本后，前端自动扫描内容。如果包含 MD 标记（`#`, `-`, `>`, `|`），AI 对话窗主动弹出一条轻量提示。

2.  **模板选择**：
    > **✨ 检测到 Markdown 格式，需要我帮你排版吗？**
    > [通用文章]  [正式报告]  [公文通知]  [教案模板]  [科研论文]  [保持原样]

3.  **一键执行**：用户点击某个模板按钮后，编辑器自动完成所有排版操作，对话框仅回复：“✅ 已按【正式报告】格式排版。以后是不是都这样排版？”，判断用户答案，如果用户无回答，默认否。

4.  **图片处理**：若文档含图片链接，排版完成后，AI 追加询问：“📷 文档中的图片需要自动对齐吗？（等比例撑满宽度 / 窄边距居中）”。选择后，所有图片按规则调整到位。

### 3. 各模板的排版规则

**通用文章**：适合日常写作和自媒体，正文用标准段落，图片窄边距居中。
-   **标题**：一级标题居中，二级及以下左对齐。
-   **正文**：首行缩进 2 字符（可选）。
-   **图片**：窄边距居中（默认占页面 80% 宽度）。

**正式报告**：适合商务和汇报，标题和正文层级分明。
-   **标题**：一级标题居中加粗，二级标题左对齐加粗，三级标题左对齐不加粗。
-   **正文**：首行缩进 2 字符。
-   **列表**：保留项目符号，使用标准缩进。
-   **图片**：居中对齐，自适应宽度。

**公文通知**：仿机关红头文件风格。
-   **标题**：全文一级标题用方正小标宋或思源宋体加粗，字号 22pt 居中。
-   **正文**：两端对齐，首行缩进 2 字符，行间距 28 磅。
-   **落款**：日期右对齐。

**教案模板**：结构化教育模板。
-   **标题**：主标题黑体二号居中。
-   **模块标题**（如“教学目标”、“重难点”）左对齐加粗。
-   **正文**：首行缩进 2 字符。

**科研论文**：符合学术规范。
-   **摘要**与**关键词**：两端对齐，左右缩进 2 字符。
-   **章节标题**（一、二、三）左对齐加粗。
-   **参考文献**：自动生成编号列表，悬挂缩进。

### 4. 图片处理规则

排版完成后，若用户选择自动对齐：

1.  **宽图（横版）**：等比缩放至文本区域最大宽度，居中显示。
2.  **长图（竖版）**：等比缩放至文本区域宽度的 65%，居中显示，上下留白。
3.  **小图（不足 50% 宽度）**：保持原尺寸，居中对齐。

### 5. 技术实现

这是一个纯前端特性，属于**本地指令的扩展**。在 `src/data/local-commands.js` 中注册为：

```javascript
{
  id: "format-md",
  patterns: [/^(排版|格式化|美化|清洗格式|一键排版)$/],
  condition: (ctx) => ctx.hasMarkdownContent,
  action: (ctx) => showTemplateSelector(ctx)
}
```

核心引擎 `useMarkdownFormatter.js` 的工作原理是：

1.  遍历文档的所有 Tiptap 节点。
2.  根据节点类型（标题、段落、列表、表格）和当前模板规则，修改节点的 `attrs`（如 `textAlign`, `fontSize`, `lineHeight`）。
3.  对图片节点，调用图片处理规则调整 `width` 和 `align`。
4.  最后通过 Tiptap 的 `setNode` 批量提交更新，一步到位。

"MD 智能排版引擎"：

1. 创建 src/composables/useMarkdownFormatter.js，导出 formatDocument(template) 函数。
   - 支持 5 个模板：article, report, official, lesson-plan, paper。
   - 每个模板定义好格式规则（标题对齐、字号、加粗、行距、图片宽度等）。
   - 遍历编辑器 JSON 内容，按规则修改每个节点的 attrs，批量更新。

2. 在 src/components/ai/LocalCommandMessage.vue 中增加模板选择器 UI。
   - 当检测到 MD 内容时，显示一排模板按钮。
   - 用户点击按钮后调用 formatDocument，然后回复"✅ 已按【XXX】格式排版"。

3. 在排版完成后，如果文档包含图片，追加询问"图片需要自动对齐吗？"并提供两个选项。

4. 注册为本地指令，关键词包括"排版"、"格式化"、"美化"等。

5. 确保所有操作在本地完成，不调用任何 AI API。
```

这样，WPX 的“智能”就不只是依赖大模型，这种懂用户痛点的规则引擎，反而是最贴心的智能。
