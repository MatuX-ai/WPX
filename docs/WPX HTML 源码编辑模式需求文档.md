### WPX HTML 源码编辑模式需求文档（草稿）

**版本**：V1.0  
**关联文档**：网页文件导入与智能排版需求文档

#### 1. 概述
为导入的网页文档提供“源码编辑模式”。用户在正常可视化编辑界面，可一键呼出左侧 HTML 源码面板，实现**左侧改代码，右侧看效果**的实时同步编辑。此设计践行了 WPX“非侵入式”原则，不给普通用户增加认知负担。

#### 2. 交互流程

1.  **触发显示**：用户导入 HTML 文件后，在编辑器顶部导航栏出现代码图标按钮。点击后，左侧滑出 HTML 源码面板，按钮保持高亮状态。
2.  **分屏编辑**：界面自动变为左右分屏。用户在左面板修改代码，右侧可视化区域**实时**渲染更新。
3.  **关闭隐藏**：用户再次点击按钮，或点击面板关闭按钮，左侧面板收起，导航栏按钮恢复默认状态。
4.  **数据同步**：源码的修改会自动更新到我们之前定义的 `attrs.htmlSource` 隐藏属性中，确保切换不同排版模式时数据一致。

#### 3. 技术实现

-   **UI 布局**：使用弹性布局，左侧源码面板宽度可拖拽调整。
-   **编辑器**：左侧嵌入轻量级代码编辑器（如 CodeMirror 或 Monaco Editor），支持 HTML 语法高亮、行号显示和括号匹配。
-   **渲染机制**：源码变化后通过防抖处理（如 300ms 延迟），将代码重新解析并更新右侧 Tiptap 编辑器的内容。为防止内容跳动，会尽量保持用户的当前滚动位置。

#### 4. 验收标准

1.  【显示/隐藏】点击 `< >` 按钮，源码面板正常滑出/收起，图标状态正确切换。
2.  【实时渲染】在左侧修改文字并稍等片刻，右侧可视化区域内容同步更新。
3.  【代码高亮】源码编辑区域支持 HTML 语法高亮。
4.  【数据持久化】修改代码后，文档属性中的 `htmlSource` 同步更新，重开文档后代码保持修改后的状态。
5.  【普通用户无扰】未导入 HTML 的普通文档，导航栏上不显示此按钮。
