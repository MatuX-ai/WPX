## 1. 概述

WPX 桌面端内置 7za 命令行工具，为用户提供无需安装第三方软件的压缩/解压缩能力。用户可在编辑器、文库、右键菜单中直接打包文档、解压资料包，延续“一站式工作站”的产品定位。

**核心差异化**：传统办公软件大多不支持或仅支持 ZIP，WPX 内置 7-Zip 引擎，默认使用 7z 格式，压缩比高 30-70%，且支持 AES-256 加密。

## 2. 内置 7za 的技术策略

### 2.1 二进制获取

| 平台 | 文件 | 大小 | 来源 |
|:---|:---|:---|:---|
| Windows (x64) | `7za.exe` | ~1.2MB | https://www.7-zip.org/ |
| macOS (arm64/x64) | `7za` | ~800KB | Homebrew 或官方编译 |
| Linux (x64) | `7za` | ~800KB | p7zip-full 包或官方编译 |

### 2.2 存放位置

Electron 打包时，将对应平台的 7za 二进制放入应用的 `resources/bin/` 目录。主进程启动时检测平台，确定可执行文件路径：

```
应用安装目录/
├── WPX.exe
├── resources/
│   ├── app.asar
│   └── bin/
│       └── 7za.exe (Windows)
│          或 7za (macOS/Linux)
```

### 2.3 许可证合规

- 7-Zip 使用 LGPL 许可证，分发二进制需在应用的“关于”页面或安装目录中包含 `7-Zip-LICENSE.txt`，并注明源码获取地址。
- WPX 不修改 7za 源码，通过 CLI 动态调用，符合 LGPL 要求。
- 在设置 → 关于页面底部添加：“本软件集成了 7-Zip 压缩引擎，7-Zip 版权归 Igor Pavlov 所有，采用 LGPL 许可证。”

### 2.4 macOS 代码签名

内置的 7za 二进制需与主应用一起签名，否则可能被 Gatekeeper 拦截：
```
codesign --force --sign "Developer ID Application: WPX" resources/bin/7za
```
首次构建时，若未配置开发者证书，先在开发模式中跳过签名验证。

## 4. 技术实现

### 4.1 核心模块

**`electron/zip-service.js`**：
- 管理 7za 二进制路径（根据平台和 dev/prod 模式确定）
- 封装 `compress()`、`extract()`、`list()` 方法
- 通过 `child_process.spawn()` 执行命令，实时解析进度输出
- 支持取消（发送 SIGTERM 或 Windows 上的 taskkill）

### 4.2 IPC 通道

| 通道 | 方向 | 参数 | 返回 |
|:---|:---|:---|:---|
| `zip:compress` | 渲染→主 | 源路径[], 输出路径, 格式, 密码, 级别 | 成功/失败 |
| `zip:extract` | 渲染→主 | 压缩包路径, 目标路径, 密码, 文件过滤[] | 成功/失败 |
| `zip:list` | 渲染→主 | 压缩包路径 | 文件列表 JSON |
| `zip:progress` | 主→渲染 | { operationId, percent, currentFile } | - |
| `zip:cancel` | 渲染→主 | operationId | - |

### 4.3 关键命令

```bash
# 压缩为 7z，极限压缩，带密码
7za a -t7z -mx=9 -p"password" output.7z /path/to/files

# 压缩为 zip，标准压缩
7za a -tzip -mx=5 output.zip /path/to/files

# 解压到指定目录
7za x archive.7z -o/path/to/output -p"password"

# 列出内容（机器可读格式 -slt）
7za l -slt archive.7z

# 显示进度（-bsp1 输出到 stdout）
7za a -bsp1 output.7z /path/to/files
```

### 4.4 进度解析

7za 的 `-bsp1` 输出格式：
```
0%
24%
57%
100%
```
每行一个百分比数字，用正则 `/(\d+)%/` 提取即可。
