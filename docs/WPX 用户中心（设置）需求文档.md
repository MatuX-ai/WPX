**状态**：定稿

## 1. 概述

用户中心是 WPX 的控制中枢，用户在此管理 AI 行为、模型配置、Skills 开关、字体与 Token、个人偏好等。设计原则：**一切可配置，默认即最佳**。

**核心功能模块**：

- **Agent 配置**：定义 AI 助手的身份、语气、专业领域。

- **Skills 库**：内置丰富技能，用户可启用/禁用，让 AI 精准调用能力。

- **模型配置**：支持接入第三方开源大模型（薅羊毛），或直接使用 WPX 平台模型。

- **字体与 Token**：管理已下载字体、查看 Token 余额和消费记录。

- **通用偏好**：主题、语言、默认保存路径等。

## 2. 功能模块详述

### 2.1 Agent 设置

用户可以为右下角的 AI 助手定义“人设”和“能力范围”。

| 配置项 | 说明 | 默认值 |

|:---|:---|:---|

| **助手名称** | 自定义显示在对话窗标题栏的名字 | “WPX 助手” |

| **身份描述** | 一句话定义助手角色，如“专业的技术文档撰写者” | “全能的写作伙伴” |

| **语气风格** | 下拉选择：专业正式、轻松口语化、幽默风趣、自定义 | 专业正式 |

| **专业领域** | 多选标签：技术开发、产品管理、市场营销、学术论文、小说创作、法律文书等 | 全部 |

| **回复长度偏好** | 简洁、标准、详细 | 标准 |

| **语言偏好** | 与用户交流的语言（中文/英文/自动） | 中文 |

这些配置会被注入到 AI 对话的 System Prompt 中，让所有 Skills 都遵循统一人设。

---

### 2.2 Skills 库

Skills 是 AI 助手的具体能力单元。WPX 内置丰富的 Skills，用户可按需启用。

#### 2.2.1 Skills 列表（MVP 阶段内置 15+）

**写作类**

- **续写**：根据上文智能续写内容。

- **改写**：选中文本后改写，支持改变语气、风格。

- **扩写**：将简短内容扩展为详细段落。

- **缩写**：将长文精简为核心要点。

- **翻译**：中英互译，保留原文格式。

- **总结**：为全文或选中段落生成摘要。

- **大纲生成**：根据主题生成结构化大纲。

- **标题生成**：为文章生成多个备选标题。

**编辑类**

- **排版美化**：自动调整段落间距、缩进、标题层级。

- **格式转换**：列表与段落互转、表格与文本互转。

- **图片处理**：调用图片处理能力（去背景、裁剪、调色等）。

- **表格操作**：智能生成表格、调整表格结构。

**知识类**

- **问答**：基于资料库或通用知识回答问题。

- **代码解释**：解释选中的代码片段。

- **数据洞察**：对选中的表格数据进行分析和总结。

#### 2.2.2 Skills 管理界面

- 每个 Skill 卡片显示：图标、名称、简介、启用/禁用开关。

- 分类筛选：[全部] [写作] [编辑] [知识]。

- 用户禁用的 Skill 不会出现在 AI 的能力列表中，AI 也不会尝试调用。

- 未来支持用户自定义 Skill（通过 Prompt 模板）。

### 2.3 我的模型

用户可自定义 AI 模型来源，既省钱又灵活。**访客与登录用户均可配置自定义 API**；WPX 公共大模型仅登录用户可用。

#### 2.3.1 页面文案

- **标题**：我的模型

- **说明**：配置你自己的大模型 API，或使用 WPX 提供的公共模型

- **API Key 提示**：你的 API Key 仅加密存储在本机，不会上传

#### 2.3.2 模型来源

| 来源 | 访客 | 登录用户 | 说明 |
|:---|:---|:---|:---|
| **WPX 公共大模型** | 不可选（隐藏） | 可选 | 消耗 WPX 每日 Token 额度或账户 Token；默认建议 100M Token/天 |
| **用户自定义模型** | 可选（唯一选项） | 可选 | 填写 Endpoint、API Key、模型名称；兼容 OpenAI 接口 |

#### 2.3.3 配置项

**文本模型**

- 自定义配置：

  - API 地址（Endpoint）

  - API Key（本地 AES 加密存储）

  - 模型名称（如 `deepseek-chat`）

- **测试连接**：发送简单请求验证配置；成功显示绿色对勾，失败显示红色叉号与错误信息

**图片识别模型**

- 配置项同文本模型

**模型参数**

- Temperature（创造性）：滑块 0-2，默认 0.7

- Top P：滑块 0-1，默认 0.9

- 最大输出长度：数字输入，默认 4096

#### 2.3.4 优先级与配额规则

- **调用优先级**：用户自定义模型 > WPX 公共大模型。

- 自定义模型**不消耗** WPX 公共 Token 额度。

- 公共大模型配额按 **Token** 计量（输入 + 输出），**不按调用次数**；每次对话完成后按模型返回的 usage 扣减。

- 若自定义模型调用失败（网络错误、余额不足等），**已登录用户**可自动回退到 WPX 公共大模型；访客无公共模型可回退。

- 在 AI 对话窗的标题栏显示当前使用的模型名称。

#### 2.3.5 隐私提醒

- 使用自定义模型时，数据将发送到用户指定的第三方服务器。

- 界面显示提示：数据将发送到第三方模型服务，WPX 不会存储您的 API Key（仅本地加密存储）。

## 4. 技术实现要点

### 4.1 数据存储

- **用户偏好**：存储在主进程的 `electron-store` 中，跨窗口同步。

- **自定义模型配置**：API Key 使用 AES 加密后存储在本地，不解密时不暴露。

- **Skills 开关状态**：存储在本地，可与账户同步（未来云同步功能）。

### 4.2 System Prompt 动态组装

AI 对话的 System Prompt 由以下部分动态拼装：

1. Agent 人设（身份、语气、专业领域、回复长度）

2. 启用的 Skills 列表及调用规范

3. 当前编辑器的上下文（选中文本、全文摘要、光标位置等）

4. 用户偏好（语言、称呼习惯等）

### 4.3 模型切换与 Token 配额

- 在 `useAiChat.js` 中读取模型配置，动态设置 API endpoint 与 headers；**自定义模型优先**。

- 使用 WPX 公共模型时：**按 Token 扣减**每日免费额度（非按次数）；对话 `onFinish` 时根据 usage 累加已用 Token。

- 调用前检查剩余 Token > 0；额度存储于 Electron `free-quota` store（按用户 ID，每日重置）。

- 已登录用户自定义模型失败时可回退公共模型；访客无公共模型回退。

5. 自定义模型优先；已登录用户自定义失败时可回退公共模型。 ✅

6. 自定义 API Key 加密存储，设置页面不直接显示完整 Key。 ✅

7. 字体管理、Token 余额、消费记录可在设置中查看。 ✅

8. 主题切换、语言切换、自动保存等通用设置正常生效。 ✅

9. 清除记忆数据后，智能模板恢复默认状态。 ✅
