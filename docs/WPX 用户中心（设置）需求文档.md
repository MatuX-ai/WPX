## 1. 概述

用户中心是 WPX 的控制中枢，用户在此管理 AI 行为、模型配置、Skills 开关、字体管理、个人偏好等。设计原则：**一切可配置，默认即最佳**。

> **V1.1 完全免费模式说明**：WPX 不再提供平台内置大模型，亦不经营商业字体 / Token 售卖业务。所有 AI 能力、字体支持均由用户在桌面端自行配置或导入。

**核心功能模块**：

- **Agent 配置**：定义 AI 助手的身份、语气、专业领域。

- **Skills 库**：内置丰富技能，用户可启用/禁用，让 AI 精准调用能力。

- **模型配置**：用户在「我的模型」中配置**第三方大模型 API**（兼容 OpenAI Chat Completions 协议），API Key 仅本地加密存储。

- **字体管理**：管理内置字体、下载在线免费字体、导入本地字体（仅限用户已获合法授权的字体文件）。

- **通用偏好**：主题、语言、默认保存路径等。

## 2. 功能模块详述

### 2.1 Agent 设置

用户可以为右下角的 AI 助手定义“人设”和“能力范围”。

| 配置项 | 说明 | 默认值 |

|:---|:---|:---|

| **助手名称** | 自定义显示在对话窗标题栏的名字 | “WPX 助手” |

| **身份描述** | 一句话定义助手角色，如“专业的技术文档撰写者” | “全能的写作伙伴” |

| **语气风格** | 下拉选择：专业正式、轻松口语化、幽默风趣、自定义 | 专业正式 |

| **专业领域** | 多选标签：技术开发、产品管理、市场营销、学术论文、小说创作、法律文书等 | 全部 |

| **回复长度偏好** | 简洁、标准、详细 | 标准 |

| **语言偏好** | 与用户交流的语言（中文/英文/自动） | 中文 |

这些配置会被注入到 AI 对话的 System Prompt 中，让所有 Skills 都遵循统一人设。

---

### 2.2 Skills 库

Skills 是 AI 助手的具体能力单元。WPX 内置丰富的 Skills，用户可按需启用。

#### 2.2.1 Skills 列表（MVP 阶段内置 15+）

**写作类**

- **续写**：根据上文智能续写内容。

- **改写**：选中文本后改写，支持改变语气、风格。

- **扩写**：将简短内容扩展为详细段落。

- **缩写**：将长文精简为核心要点。

- **翻译**：中英互译，保留原文格式。

- **总结**：为全文或选中段落生成摘要。

- **大纲生成**：根据主题生成结构化大纲。

- **标题生成**：为文章生成多个备选标题。

**编辑类**

- **排版美化**：自动调整段落间距、缩进、标题层级。

- **格式转换**：列表与段落互转、表格与文本互转。

- **图片处理**：调用图片处理能力（去背景、裁剪、调色等）。

- **表格操作**：智能生成表格、调整表格结构。

**知识类**

- **问答**：基于资料库或通用知识回答问题。

- **代码解释**：解释选中的代码片段。

- **数据洞察**：对选中的表格数据进行分析和总结。

#### 2.2.2 Skills 管理界面

- 每个 Skill 卡片显示：图标、名称、简介、启用/禁用开关。

- 分类筛选：[全部] [写作] [编辑] [知识]。

- 用户禁用的 Skill 不会出现在 AI 的能力列表中，AI 也不会尝试调用。

- 未来支持用户自定义 Skill（通过 Prompt 模板）。

**文本模型**

## 4. 技术实现要点

### 4.1 数据存储

- **用户偏好**：存储在主进程的 `electron-store` 中，跨窗口同步。

- **自定义模型配置**：API Key 使用 AES 加密后存储在本地，不解密时不暴露。

- **Skills 开关状态**：存储在本地，可与账户同步（未来云同步功能）。

### 4.2 System Prompt 动态组装

AI 对话的 System Prompt 由以下部分动态拼装：

1. Agent 人设（身份、语气、专业领域、回复长度）

2. 启用的 Skills 列表及调用规范

3. 当前编辑器的上下文（选中文本、全文摘要、光标位置等）

4. 用户偏好（语言、称呼习惯等）

### 4.3 模型切换与配额

> **V1.1 变更**：由于不再提供 WPX 公共大模型服务，「公共 Token 额度」「按 Token 扣减」「回退公共模型」等逻辑**全部下线**。V1.1 仅保留「用户自定义模型」调用路径。

- 在 `useAiChat.js` 中读取模型配置，动态设置 API endpoint 与 headers；**仅使用用户自定义模型**。

- ~~使用 WPX 公共模型时：按 Token 扣减每日免费额度（非按次数）；对话 `onFinish` 时根据 usage 累加已用 Token。~~（已废弃）

- ~~调用前检查剩余 Token > 0；额度存储于 Electron `free-quota` store（按用户 ID，每日重置）。~~（已废弃）

- 自定义模型调用失败时，返回明确错误（余额不足 / Key 无效 / 网络异常等），**不再回退到任何平台模型**。
