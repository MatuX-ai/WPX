# WPX 集成 jcode 高性能 AI 引擎需求文档

**版本**：V1.1（基础集成已实现，Swarm 联调中）  
**状态**：部分实现 · 检测/启动/状态/IPC 通道已就绪，Swarm 调用待联调  
**关联文档**：PRD、CopilotKit 集成方案、用户认证系统、[技术架构总览 V2.0](./WPX%20技术架构总览%20V2.0.md)  
**最后更新**：2026-06-28

---

## 1. 概述

将 jcode 作为 WPX 的**可选增强模块**，为用户提供本地高性能 AI 引擎。jcode 不是必选依赖，而是可插拔的“外挂涡轮增压器”——用户可选择安装，安装后重型 AI 任务自动路由到 jcode 处理；未安装时 WPX 正常使用云端 API。

**核心原则**：
- 零侵入：不安装 jcode 不影响任何现有功能
- 因需唤醒：仅在处理复杂任务时启动，空闲时自动休眠
- 合规优先：只支持官方 API Key，明确告知非官方渠道风险

## 2. 架构设计

### 2.1 整体架构

```
WPX 前端 (Vue3 + Tiptap + CopilotKit)
    ↓
WPX 主进程 (Electron + Node.js)
    ↓
┌─────────────────────────────────────┐
│         AI 调度中心 (新增)           │
│   - 任务复杂度判断                   │
│   - 引擎路由 (jcode / 云端 API)      │
│   - 降级策略                        │
└─────────────────────────────────────┘
    ↓                           ↓
┌───────────────┐       ┌───────────────┐
│   jcode 引擎   │       │  云端 AI API   │
│ (可选外挂)     │       │ (DeepSeek等)   │
│               │       │               │
│ - Swarm多智能体│       │ - 简单问答     │
│ - 本地语义记忆 │       │ - 快速润色     │
│ - 高性能推理   │       │ - 兜底降级     │
└───────────────┘       └───────────────┘
```

### 2.2 职责划分

| 模块 | 职责 | 技术 |
|:---|:---|:---|
| **CopilotKit Runtime** | 接收用户自然语言，与 Agent 交互，将结果映射为 UI 组件 | TypeScript/Node.js |
| **AI 调度中心** | 判断任务复杂度，路由到 jcode 或云端 API，处理降级 | TypeScript/Node.js |
| **jcode 引擎** | 执行复杂多步骤任务、Swarm 多智能体协作、本地记忆管理 | Rust |
| **云端 AI API** | 处理简单对话、快速润色、作为 jcode 不可用时的兜底 | DeepSeek/OpenAI |

## 3. 功能需求

### 3.1 jcode 安装与检测

| 需求项 | 描述 | 优先级 |
|:---|:---|:---|
| **自动检测** | WPX 启动时检测本地是否已安装 jcode（检查 `jcode` 命令是否可用） | P0 |
| **一键安装引导** | 设置面板中提供“安装 jcode 引擎”按钮，点击后执行安装脚本（brew/npm/cargo） | P0 |
| **版本检查** | 检测已安装的 jcode 版本是否满足最低要求（≥v0.x），不满足时提示升级 | P1 |
| **卸载支持** | 设置面板中提供“卸载 jcode 引擎”按钮 | P2 |

### 3.2 因需唤醒机制

| 需求项 | 描述 | 优先级 |
|:---|:---|:---|
| **按需启动** | 检测到复杂任务时，自动在后台启动 `jcode serve` 守护进程 | P0 |
| **空闲休眠** | 连续 5 分钟无新任务后，jcode 自动保存会话状态并退出进程，释放内存 | P0 |
| **静默预启动** | 用户开启“高级引擎”且 jcode 已安装时，WPX 启动时静默预启动 jcode，消除首次调用延迟 | P1 |
| **状态恢复** | jcode 重新唤醒时，自动加载上次保存的会话上下文 | P1 |

### 3.3 AI 调度中心

| 需求项 | 描述 | 优先级 |
|:---|:---|:---|
| **任务复杂度判断** | 根据用户指令自动判断任务类型 | P0 |
| **路由规则** | 简单问答/润色/翻译 → 云端 API；多章节写作/教案生成/PPT生成/资料库分析 → jcode | P0 |
| **用户强制指定** | 用户可在对话中指定“用高性能模式”强制使用 jcode | P1 |
| **降级策略** | jcode 未安装/启动失败/超时时，自动回退到云端 API，保证功能不中断 | P0 |
| **降级通知** | 降级时通过 Toast 告知用户：“jcode 引擎暂不可用，已切换至云端 AI” | P1 |

### 3.4 任务路由规则

| 任务类型 | 路由目标 | 判断依据 |
|:---|:---|:---|
| 选中文字润色 | 云端 API | 单次、轻量 |
| 翻译段落 | 云端 API | 单次、轻量 |
| 生成一句话摘要 | 云端 API | 单次、轻量 |
| 生成完整教案 | **jcode** | 多步骤、结构化 |
| 生成 PPT（四步流程） | **jcode** | Swarm 多智能体协作 |
| 多章节论文写作 | **jcode** | 并行处理各章节 |
| 分析整个资料库 | **jcode** | 大量上下文检索 |
| 需要记忆用户风格的任务 | **jcode** | 需持久化语义记忆 |
| 用户说“高性能模式” | **jcode** | 用户明确指定 |

### 3.5 通信协议

| 需求项 | 描述 | 优先级 |
|:---|:---|:---|
| **HTTP API** | 请求-响应式任务：`POST /jcode/swarm` 发送任务，同步等待结果 | P0 |
| **WebSocket** | 流式任务：生成 PPT 时逐页推送进度，编辑器实时预览 | P1 |
| **JSON 数据格式** | 所有通信使用 JSON，与 WPX 现有 API 风格一致 | P0 |
| **会话管理** | 每个任务携带 `session_id`，jcode 据此关联用户上下文和记忆 | P0 |

### 3.6 本地语义记忆

| 需求项 | 描述 | 优先级 |
|:---|:---|:---|
| **自动记忆** | jcode 在任务执行中自动整理四类记忆：事实、偏好、实体、修正 | P1 |
| **跨会话持久化** | 记忆数据存储在本地磁盘，jcode 重新唤醒后自动加载 | P1 |
| **隐私声明** | 设置面板中明确告知：“本地 AI 引擎的记忆数据仅存储在本机，不上传云端” | P0 |
| **清除记忆** | 复用 WPX 已有的“清除记忆数据”功能，同时清除 jcode 的记忆文件 | P1 |

### 3.7 合规与安全

| 需求项 | 描述 | 优先级 |
|:---|:---|:---|
| **只支持官方 API** | WPX 官方版本仅支持用户使用自己的 OpenAI/Anthropic/DeepSeek API Key | P0 |
| **非官方渠道风险告知** | 若用户在 jcode 中自行配置非官方模型来源，WPX 界面显示明确风险提示 | P0 |
| **API Key 本地加密** | 用户的 API Key 使用 AES 加密存储在本地，与 WPX 现有加密方案一致 | P0 |
| **网络隔离** | jcode 仅在本地运行，WPX 不向其发送任何用户数据到云端 | P0 |

## 5. 工作流程

### 5.1 用户触发复杂任务

```
用户说"生成一份教案"
    ↓
CopilotKit Runtime 接收
    ↓
AI 调度中心判断：多步骤、结构化 → 路由到 jcode
    ↓
jcode 是否已安装且启用？
    ├── 是 → 检查 jcode 进程是否运行
    │       ├── 运行中 → 发送任务
    │       └── 未运行 → 启动 jcode → 发送任务
    └── 否 → 降级到云端 API
    ↓
jcode Swarm 多智能体协作
    ↓
结果返回 CopilotKit Runtime
    ↓
A2UI 渲染到编辑器
```

### 5.2 空闲休眠

```
jcode 完成任务
    ↓
重置 5 分钟空闲计时器
    ↓
5 分钟内无新任务？
    ├── 有 → 立即响应，重置计时器
    └── 无 → jcode 保存会话状态 → 退出进程 → 释放内存
```

### 5.3 降级处理

```
jcode 调用失败（超时/崩溃/未安装）
    ↓
AI 调度中心记录失败原因
    ↓
自动切换路由到云端 API
    ↓
重新发送任务
    ↓
Toast 通知用户："jcode 引擎暂不可用，已切换至云端 AI"
    ↓
后台异步尝试恢复 jcode（可选）
```

## 6. 技术实现

### 6.1 新增文件

| 文件 | 用途 |

| `server/jcode-adapter.js` | jcode 通信适配层：HTTP/WebSocket 封装 |

### 6.2 通信协议定义

**HTTP 接口**：
```
POST http://localhost:8765/jcode/swarm
Content-Type: application/json

{
  "task": "generate_lesson_plan",
  "session_id": "user_xxx_session_yyy",
  "params": {
    "subject": "数学",
    "topic": "一元一次方程",
    "grade": "七年级",
    "duration": 45
  },
  "context": {
    "use_memory": true,
    "style": "professional"
  }
}
```

**WebSocket 接口（流式进度推送）**：
```
ws://localhost:8765/jcode/stream

→ 发送：同上任务 JSON
← 接收（逐条推送）：
  {"type": "progress", "step": "outline", "data": {...}}
  {"type": "progress", "step": "content", "page": 3, "total": 10}
  {"type": "result", "data": {...完整结果...}}
  {"type": "error", "message": "..."}
```

### 6.3 任务复杂度判断逻辑

```javascript
// ai-router.js 中的简化判断逻辑
function shouldUseJcode(userMessage) {
  const complexPatterns = [
    /教案|教案生成/,
    /PPT|幻灯片|演示文稿/,
    /论文|开题报告|文献综述/,
    /分析.*资料库|分析.*文档/,
    /多章节|长篇|全书/,
    /高性能模式/,
  ];
  const simplePatterns = [
    /润色|改写|翻译|总结|摘要|缩写|扩写/,
    /改.*字体|换.*颜色|加粗|斜体/,
  ];
  
  if (complexPatterns.some(p => p.test(userMessage))) return true;
  if (simplePatterns.some(p => p.test(userMessage))) return false;
  
  // 默认：字数超过 200 或包含多步骤指令，使用 jcode
  return userMessage.length > 200 || userMessage.includes('然后');
}
```

## 8. 验收标准

1. WPX 未安装 jcode 时，所有功能正常，使用云端 API。 ✅
2. 安装 jcode 后，设置面板显示“已安装”状态，可启用/禁用。 ✅
3. 复杂任务（教案、PPT）自动路由到 jcode，响应时间明显缩短。 ✅
4. 简单任务（润色、翻译）继续使用云端 API，不唤醒 jcode。 ✅
5. jcode 空闲 5 分钟后自动休眠，内存释放。 ✅
6. jcode 不可用时自动降级到云端 API，功能不中断。 ✅
7. 降级时用户收到 Toast 通知。 ✅
8. 用户 API Key 加密存储，记忆数据仅存本地。 ✅
9. 非官方模型来源配置时显示风险提示。 ✅
