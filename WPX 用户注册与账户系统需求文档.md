# WPX 用户注册与账户系统需求文档

**版本**：V1.0  

**状态**：定稿  

**关联文档**：PRD、用户中心（设置）、字体库需求、部署域名

---

## 1. 概述

WPX 采用**访客优先、按需注册**的认证策略。用户无需注册即可使用大部分功能，仅在需要调用 WPX 公共大模型或购买字体 Token 时才需登录。注册用户可享受更完整的 AI 能力和商业字体授权。

**核心原则**：

- 不强制注册，工具免费，算力与内容付费。

- 访客数据完全本地，注册后可选择云同步。

- 对接统一认证服务 `account.proclaw.cc`。

---

## 2. 两种用户状态

### 2.1 访客（未登录）

| 维度 | 说明 |

|:---|:---|

| **标识** | 本地生成匿名设备 ID，无个人身份 |

| **可用功能** | 全部编辑器、表格、图片处理、skillhub 免费 Skills、内置免费字体、自导入字体 |

| **AI 能力** | 使用后台配置的**免费公共大模型**（通过 `ai.proclaw.cc` 的免费额度），有每日调用次数限制 |

| **自定义模型** | 不可见配置入口，不可添加自己的 API Key |

| **字体 Token** | 不可购买，不可使用 WPX 字体商店商业字体导出 |

| **数据存储** | 全部存储在本地（Electron `userData`），不清除缓存则保留 |

| **跨设备** | 不支持 |

### 2.2 注册用户（已登录）

| 维度 | 说明 |

|:---|:---|

| **标识** | 通过 `account.proclaw.cc` 登录，获取 JWT token，绑定用户 ID |

| **可用功能** | 访客全部功能 + 以下增强 |

| **AI 能力** | 可使用 WPX 公共大模型（消耗 Token 或免费额度），也可添加自己的 API Key 使用第三方模型 |

| **自定义模型** | 可配置多个自定义大模型，加密存储 API Key |

| **字体 Token** | 可充值购买 Token，使用 WPX 字体商店商业字体导出 |

| **数据存储** | 本地存储 + 可选的云端同步（未来） |

| **跨设备** | 登录后偏好和设置可同步（未来） |

---

## 3. 认证流程

### 3.1 应用启动

1. 应用启动，检查本地是否存在有效 JWT。

2. 有有效 JWT → 调用 `account.proclaw.cc` 校验 token 有效性。

   - 有效 → 进入登录状态，加载用户偏好。

   - 无效 → 清除本地 token，进入访客模式。

3. 无 token → 进入访客模式，生成或读取本地匿名设备 ID。

### 3.2 登录

1. 用户点击界面上的“登录/注册”按钮（TitleBar 或 AI 对话窗顶部）。

2. 调用系统浏览器打开 `https://account.proclaw.cc/login?callback=wpx://auth`[。](https://account.proclaw.cc/login?callback=wpx://auth`。)

3. 用户在认证页面完成登录或注册（支持邮箱/手机/第三方）。

4. 认证成功，页面重定向到 `wpx://auth?token=xxx&refresh_token=xxx`。

5. WPX 通过自定义协议或本地 HTTP 回调接收 token，存储到本地加密存储。

6. 应用切换为登录状态，加载用户偏好和 Token 余额。

### 3.3 登出

1. 用户点击“退出登录”。

2. 清除本地存储的 JWT 和用户偏好缓存。

3. 恢复为访客模式，AI 模型回退为免费公共大模型。

4. 本地文档和字体保留，不受影响。

### 3.4 Token 刷新

- JWT 短期有效（如 2 小时），refresh_token 长期有效（如 30 天）。

- 每次调用 `account.proclaw.cc` 的 API 时，若 token 即将过期，自动用 refresh_token 换取新 token。

- 刷新失败则强制登出，提示重新登录。

---

## 4. 功能限制与引导

### 4.1 免费大模型调用限制

- 后台配置每日免费调用次数（如 50 次/天），按设备 ID 计数（访客）或用户 ID（登录用户的基础免费额度）。

- 次数用尽后，AI 对话返回提示：“今日免费次数已用完，请明天再试或登录使用更多额度。”

- 登录用户额外拥有基础免费额度，仍用尽可购买 Token 调用 WPX 大模型。

### 4.2 引导登录的场景

| 触发点 | 引导语 |

|:---|:---|

| 模型配置页 | “登录后可使用自己的大模型 API Key，或购买 Token 使用 WPX 强大模型” |

| 字体商店 | “登录后可购买 Token，使用商业字体导出” |

| Token 充值页 | “登录后可充值 Token，管理余额” |

| 免费次数用尽 | “登录后获得更多免费次数，或使用自带模型” |

| Skills 中需要登录的 | “该高级技能需要登录后使用” |

## 5. 对接 [account.proclaw.cc](http://account.proclaw.cc) 接口

| 接口 | 方法 | 说明 |

|:---|:---|:---|

| `/api/auth/login` | POST | 邮箱/手机 + 密码登录，返回 token |

| `/api/auth/register` | POST | 注册新账号 |

| `/api/auth/refresh` | POST | 刷新 token |

| `/api/auth/logout` | POST | 登出，作废 refresh_token |

| `/api/user/profile` | GET | 获取用户基本信息（昵称、头像） |

| `/api/user/settings` | GET/PUT | 获取或更新用户偏好（云端同步） |

WPX 本地仅与 `account.proclaw.cc` 通信，其他服务（skillhub、ai、字体商店）通过 WPX 后端中转或直接调用时附带 JWT。

## 6. 数据隔离与隐私

- **访客数据**：所有文档、上传资料、对话历史存于本地，不上传服务器。

- **登录用户**：本地数据不变，用户可选择开启云端同步（未来功能，同步到 `account.proclaw.cc` 关联的存储服务）。

- **API Key 存储**：无论访客还是登录用户，自定义 API Key 均 AES 加密存储在本地，不经过 WPX 服务器。

- **Token 消费记录**：仅登录用户产生，存储在 WPX 后端，关联用户 ID。

## 7. 界面元素

### 7.1 登录状态指示

- TitleBar 右侧或 AI 对话窗顶部显示用户状态：

  - 访客：显示“未登录”文字 + 登录按钮。

  - 登录用户：显示头像（可选）+ 昵称，点击弹出“我的设置”“退出登录”菜单。

### 7.2 设置页调整

- 模型配置、字体 Token 相关页面，在访客模式下显示引导登录提示，不展示完整功能。

---

## 8. 安全考虑

- JWT 存储在 Electron 的 `safeStorage` 或 `electron-store` 加密字段。

- 自定义协议回调 `wpx://` 需注册为系统协议，防止拦截。

- 每次启动时校验 token 有效性，防止过期 token 被盗用。

- 所有到 `account.proclaw.cc` 的请求使用 HTTPS。

## 9. 验收标准

1. 首次启动应用，无需登录，进入访客模式，可正常编辑文档。 ✅

2. 访客使用 AI 对话，使用后台免费大模型，有每日次数限制。 ✅

3. 点击登录，跳转 `account.proclaw.cc`，登录成功回调，应用变为登录状态。 ✅

4. 登录后，可添加自定义 API Key，切换模型成功。 ✅

5. 登录后，可充值 Token 并使用商业字体导出。 ✅

6. 免费次数用尽，提示登录或等待次日。 ✅

7. 退出登录后，恢复到访客模式，自定义模型配置被隐藏。 ✅

8. 本地 JWT 过期后自动刷新，或提示重新登录。 ✅

9. 导入的字体、本地文档在登出后仍保留。 ✅
